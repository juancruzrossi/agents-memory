# Agents Memory

Agents Memory is a local memory system for agent sessions, focused on preserving reusable project-specific knowledge without forcing future sessions to rediscover it.

## Language

**Memory Entry**:
A reusable piece of project knowledge that may influence how future agent sessions interpret, modify, execute, or review a project.
_Avoid_: Learning as the umbrella term

**Atomic Memory Entry**:
A Memory Entry that captures exactly one reusable idea.
_Avoid_: Combined note, bundled decision

**Instruction**:
A Memory Entry that future agents are expected to follow.
_Avoid_: Rule, mandate

**Learning**:
A Memory Entry that captures stable project knowledge or user preference without necessarily commanding behavior.
_Avoid_: Session summary, note

**Observation**:
A Memory Entry that records something discovered during a session that may be useful but is not yet normative.
_Avoid_: Fact, log line

**Decision**:
A Memory Entry that captures a chosen direction and the trade-off behind it.
_Avoid_: Opinion, idea

**Active Memory Entry**:
A Memory Entry that is currently valid for a Project and eligible for Startup Injection.
_Avoid_: Current note, latest learning

**Retired Memory Entry**:
A Memory Entry kept for history but no longer eligible for Startup Injection.
_Avoid_: Deleted memory, stale instruction

**Superseded Memory Entry**:
A Memory Entry that has been replaced by a newer Memory Entry.
_Avoid_: Edited old memory, overwritten memory

**Project**:
The local work context a Memory Entry belongs to.
_Avoid_: Repo, folder, workspace

**Related Project**:
A Project whose canonical path is an ancestor, descendant, or same-path sibling of the current Project and may help users discover Memory Entries without knowing the exact Project path.
_Avoid_: Foreign project, unrelated memory

**Project Identity**:
The stable key used to match the current working directory to a Project.
_Avoid_: Complex repository fingerprint

**Global Store**:
The local persistence layer that stores project-scoped Memory Entries outside the projects themselves.
_Avoid_: Project file, repo metadata

**Global Config**:
The local configuration file for Agents Memory, stored outside all projects.
_Avoid_: Project config file

**Agent Client**:
An agent runtime that reads from or writes to the Global Store.
_Avoid_: Model, editor

**Agent**:
The agent runtime that proposed or persisted a Memory Entry.
_Avoid_: Model, AI Model

**Agent Adapter**:
A thin integration layer that lets an Agent Client use the Global Store through its native hooks, skills, or commands.
_Avoid_: Separate implementation, duplicated logic

**Internal Tool**:
The shared local executable used by Agent Adapters to access the Global Store.
_Avoid_: User-facing CLI

**Output Format**:
The representation returned by the Internal Tool for agent display, startup context, or tests.
_Avoid_: Parse-only human text

**Core Runtime**:
The implementation runtime used by the Internal Tool.
_Avoid_: Installer shell

**Timestamp**:
A UTC ISO-8601 time value generated by the Core Runtime in `YYYY-MM-DDTHH:MM:SSZ` format.
_Avoid_: Agent-generated date, locale-specific date

**Shared Skill Source**:
The single canonical set of skill directories that Agent Clients access through symlinks.
_Avoid_: Copied skill, generated duplicate

**Skill Standard**:
The required structure and style for shared skills, following the installed skill-creator guidance.
_Avoid_: Ad hoc skill file

**Safe Skill Metadata**:
Documented skill frontmatter fields used to constrain risky skill invocation behavior.
_Avoid_: Undocumented metadata, decorative metadata

**Thin Skill**:
A skill that gives strict procedural guidance while delegating business logic to the Internal Tool.
_Avoid_: Business logic skill, SQL skill

**Symlink Install**:
An installation strategy where each Agent Client receives symbolic links to the Shared Skill Source instead of copied skill files.
_Avoid_: Copy fallback

**Symlink Conflict**:
An existing skill path that does not point to the expected Shared Skill Source.
_Avoid_: Silent overwrite

**Installer**:
The interactive setup script that creates the global Agents Memory home and symlinks skills into Agent Clients.
_Avoid_: Package manager release

**Updater**:
The maintenance script that updates the global Agents Memory installation without recreating per-agent skill copies.
_Avoid_: Manual skill copy, per-agent update

**Uninstaller**:
The maintenance script that removes Agents Memory symlinks without deleting stored memories by default.
_Avoid_: Purge mode, destructive uninstall

**Project Script**:
A root-level script under the project's `scripts/` directory.
_Avoid_: Root shell script

**Memory Backup**:
A timestamped copy of the SQLite database created before maintenance operations.
_Avoid_: Migration snapshot

**Installed Agent Directory**:
An existing global configuration directory that indicates an Agent Client is installed.
_Avoid_: Directory created by installer

**Startup Injection**:
The automatic loading of relevant enabled Memory Entries into an Agent Client at session start.
_Avoid_: Inject skill, save skill

**Compact Memory View**:
A token-conscious representation of Active Memory Entries prepared for Startup Injection.
_Avoid_: Full memory dump, verbose history

**Startup Budget**:
The maximum size allowed for the Compact Memory View injected into an Agent Client.
_Avoid_: Unlimited memory context

**Memory Priority**:
The relative importance of a Memory Entry when building a Compact Memory View.
_Avoid_: Sort order, severity

**Save Proposal**:
A proposed set of Memory Entry changes produced from a session but not persisted until approved by the user.
_Avoid_: Auto-save, implicit memory write

**Save Operation**:
An individual create, supersede, retire, or keep action inside a Save Proposal.
_Avoid_: Inline edit, silent dedupe

**Partial Approval**:
User approval for only selected Save Operations in a Save Proposal.
_Avoid_: All-or-nothing save

**Get Learnings Skill**:
The user-facing skill that displays Memory Entries for the current Project.
_Avoid_: Query CLI, memory dump

**Save Learnings Skill**:
The user-facing skill that proposes Memory Entry changes from the current conversation and persists only approved changes.
_Avoid_: Autosave skill, transcript logger

**Setup Skill**:
The user-invoked skill that configures Agents Memory integrations and startup hooks for supported Agent Clients.
_Avoid_: README setup, automatic setup

**Current Agent Client**:
The Agent Client runtime from which a skill is invoked.
_Avoid_: User-selected agent

## Relationships

- A **Project** is the only scope for **Memory Entries**
- A **Related Project** may be displayed by the **Get Learnings Skill** for discovery, without changing which Project owns each Memory Entry
- A **Project Identity** prefers Git metadata when available and otherwise uses the canonical path
- A **Project** has zero or more **Memory Entries**
- A **Memory Entry** belongs to exactly one **Project**
- A **Memory Entry** has exactly one type: **Instruction**, **Learning**, **Observation**, or **Decision**
- A **Memory Entry** should be an **Atomic Memory Entry**
- An **Active Memory Entry** is eligible for **Startup Injection**
- A **Retired Memory Entry** remains in the **Global Store** for history but is not injected
- A **Superseded Memory Entry** remains in the **Global Store** and points to the newer **Memory Entry** that replaced it
- A **Global Store** contains **Memory Entries** for one or more **Projects**
- **Global Config** lives outside projects and configures shared Agents Memory behavior
- An **Agent Client** uses the **Global Store** without adding files to a **Project**
- An **Agent Adapter** connects exactly one **Agent Client** to the **Internal Tool**
- The **Internal Tool** is not the primary user experience; users interact through their **Agent Client**
- The **Internal Tool** supports text and JSON **Output Formats**
- The **Core Runtime** for the **Internal Tool** is Python 3.9 or newer
- **Timestamps** are generated consistently by the **Core Runtime**, not independently by skills
- A **Shared Skill Source** is installed into **Agent Clients** through **Symlink Installs**
- A **Shared Skill Source** must follow the **Skill Standard**
- **Safe Skill Metadata** may extend the **Skill Standard** when an Agent Client documents the field and it reduces side-effect risk
- Shared skills are **Thin Skills** with strong invocation procedures
- The **Installer** performs **Symlink Installs** and initializes the global Agents Memory home
- The **Installer** delegates to the **Updater** when it detects an existing global Agents Memory installation
- The **Installer** detects installed Agent Clients by checking their global configuration directories
- The **Installer** only targets existing **Installed Agent Directories** and must not create root directories for absent Agent Clients
- A **Symlink Conflict** requires explicit user confirmation before replacement
- The **Updater** changes the **Shared Skill Source** so symlinked Agent Clients receive skill updates transparently
- The **Updater** creates a **Memory Backup** before replacing tooling or shared skills
- The **Installer**, **Updater**, and **Uninstaller** are **Project Scripts**
- The **Uninstaller** removes expected symlinks but keeps stored memories by default
- **Startup Injection** is handled by Agent Client integration, not by a user-facing save skill
- **Startup Injection** uses a **Compact Memory View** instead of raw Memory Entries
- **Startup Injection** must fit within a **Startup Budget**
- **Memory Priority** helps decide which Active Memory Entries appear in the **Compact Memory View**
- A **Save Proposal** must be approved by the user before it changes the **Global Store**
- A **Save Proposal** contains one or more **Save Operations**
- A **Save Proposal** supports **Partial Approval**
- `supersede` and `retire` **Save Operations** target only **Active Memory Entries**
- A **Memory Entry** stores its content, rationale, and **Agent** but not the full conversation transcript
- The **Get Learnings Skill** shows only **Active Memory Entries** by default and all Memory Entries when called with `--all`
- The **Get Learnings Skill** includes **Related Project** Memory Entries so users can discover memories from a workspace root or nested directory without knowing the exact Project path
- The v1 shared skills are **Get Learnings Skill**, **Save Learnings Skill**, and **Setup Skill**
- The **Setup Skill** configures the **Current Agent Client** and must detect it instead of asking the user to name it
- The **Setup Skill** uses current official documentation when it needs to configure hooks or plugin-based startup injection for the **Current Agent Client**
- The **Setup Skill** must not configure or suggest configuration for other Agent Clients
- OpenCode uses a global plugin that injects startup memory through its documented system-transform hook
- Codex `SessionStart` hooks require camelCase JSON with `hookSpecificOutput.hookEventName` and `hookSpecificOutput.additionalContext`
- Codex rejects unknown fields in `hookSpecificOutput`; do not include `suppressOutput` there
- Codex stores per-hook trust and enablement in `~/.codex/config.toml`, and setup may enable this hook only after a `trusted_hash` already exists
- Amp remains skills-only in v1 unless a verified startup-injection mechanism is documented later

## Example Dialogue

> **Dev:** "Should this go into memory?"
> **Domain expert:** "Only if it becomes a reusable **Memory Entry** for this **Project**. If the agent must obey it, store it as an **Instruction**; if it is stable context, store it as a **Learning**."

## Flagged Ambiguities

- "learning" was originally used to mean every kind of remembered information; resolved: **Memory Entry** is the umbrella term, while **Learning** is only one type.
- user-level memory was considered; resolved: **Memory Entries** are only project-scoped because global user preferences belong in external agent instruction files such as AGENTS.md or CLAUDE.md.
- project-local metadata was considered; resolved: the **Global Store** must not add files to projects, and project identity should prefer Git metadata when available or the canonical path otherwise.
- complex Git identity normalization was considered; resolved: **Project Identity** should stay simple in the first version while remaining extensible.
- Git repositories without a remote were considered; resolved: they use canonical path **Project Identity** in the first version.
- a user-facing CLI was considered; resolved: command-line access may exist internally, but the primary experience must happen through Agent Client hooks, skills, or commands.
- copied agent integrations were considered; resolved: **Agent Clients** should use **Symlink Installs** to a **Shared Skill Source**, with installation validation rather than copy fallback.
- "inject-learnings" was considered as a skill name; resolved: injection is a startup behavior, not a user-facing skill, to avoid confusion with saving memories.
- stale project knowledge must not compete with newer project knowledge; resolved: superseded or outdated **Memory Entries** become **Retired Memory Entries** and are excluded from **Startup Injection**.
- updating project knowledge must preserve history; resolved: replacement creates a new **Active Memory Entry** and marks the previous one as a **Superseded Memory Entry**.
- automatic memory writes were considered; resolved: **Save Proposals** require explicit user approval before persistence.
- bundled project knowledge was considered; resolved: each **Memory Entry** should capture one reusable idea so it can be superseded independently.
- storing full conversation evidence was considered; resolved: persist only the atomic memory content, its rationale, and the **Agent** involved.
- storing exact model names was considered; resolved: store only the **Agent**, such as codex, claude-code, opencode, or amp.
- historical memory display was considered; resolved: **Get Learnings Skill** defaults to active entries and supports `--all` for active, retired, and superseded entries.
- exact-path lookup made workspace-root and nested-directory sessions look empty even when **Related Projects** had Memory Entries; resolved: **Get Learnings Skill** displays Related Project Memory Entries while preserving each Memory Entry's owning Project.
- duplicate memory handling was considered; resolved: **Save Operations** are limited to create, supersede, retire, or keep after comparing with existing Memory Entries.
- all-or-nothing save approval was considered; resolved: **Save Proposals** support **Partial Approval** such as approving selected operations.
- changing historical entries was considered; resolved: `supersede` and `retire` operate only on **Active Memory Entries** so historical links are preserved.
- startup context size was considered; resolved: **Startup Injection** must use a **Compact Memory View** to avoid wasting agent context tokens.
- unlimited startup memory was considered; resolved: **Startup Injection** must use a **Startup Budget** and prioritize Memory Entries by type and **Memory Priority**.
- the initial **Startup Budget** is 9000 characters.
- global configuration location was considered; resolved: **Global Config** lives at `~/.agents-memory/config.toml`, and future project-specific configuration should live in the **Global Store** rather than project files.
- package distribution was considered; resolved: the first version uses an interactive **Installer** rather than a published package.
- root-level maintenance scripts were considered; resolved: maintenance scripts live under the root `scripts/` directory.
- complex uninstall flags were considered; resolved: v1 **Uninstaller** stays simple and non-destructive by default.
- per-agent skill copies were considered; resolved: skills live in the **Shared Skill Source** under `~/.agents-memory/skills` and Agent Clients point to them through **Symlink Installs**.
- agent CLI detection was considered; resolved: the **Installer** treats an Agent Client as installed when its global configuration directory exists.
- creating missing agent directories was considered; resolved: the **Installer** must not create root directories for Agent Clients that are not already installed.
- silent symlink replacement was considered; resolved: **Symlink Conflicts** require explicit user confirmation before replacement.
- automatic SQLite migrations in update were considered; resolved: v1 does not include schema migrations beyond first-time initialization.
- updating without database protection was considered; resolved: the **Updater** creates a timestamped **Memory Backup** before changing installed tooling.
- reinstall behavior was considered; resolved: when an existing global installation is found, the **Installer** delegates to the **Updater** instead of maintaining a separate reinstall path.
- core implementation language was considered; resolved: the **Core Runtime** is Python 3.9 or newer, while the **Installer** may use Bash.
- ad hoc skill structure was considered; resolved: all shared skills must follow the installed skill-creator **Skill Standard**.
- skill-owned business logic was considered; resolved: shared skills are **Thin Skills** that prescribe behavior strongly while delegating persistence and project matching to the **Internal Tool**.
- human-only output was considered; resolved: the **Internal Tool** supports text and JSON **Output Formats** for agent display, startup injection, tests, and future integrations.
- timestamp consistency was considered; resolved: all stored dates must use one **Timestamp** format generated by the **Core Runtime** across skills and Agent Clients.
- **Timestamps** use UTC ISO-8601 format with a trailing `Z`, such as `2026-05-08T16:42:31Z`.
- README-based setup was considered; resolved: setup should be exposed as a **Setup Skill** rather than requiring the user to follow manual documentation.
- strict minimal skill frontmatter was considered; resolved: **Safe Skill Metadata** such as `disable-model-invocation: true` is allowed for setup skills when officially documented by an Agent Client.
- v1 skill surface was considered; resolved: ship `get-learnings`, `save-learnings`, and `setup-agents-memory`, and do not ship an `inject-learnings` skill.
- agent selection during setup was considered; resolved: **Setup Skill** detects and configures the **Current Agent Client** instead of requiring the user to pass an agent name.
- cross-agent setup was considered; resolved: each Agent Client configures only its own integration when **Setup Skill** is invoked from that client.
- Codex hook output shape was considered; resolved: emit the official camelCase `SessionStart` payload and keep `hookSpecificOutput` free of unsupported fields.
- Codex hook trust flow was considered; resolved: setup may repair `enabled = false` only when Codex has already recorded a `trusted_hash` for the exact hook.
